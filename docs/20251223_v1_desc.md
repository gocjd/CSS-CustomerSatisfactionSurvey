## 시스템 개요

### 목적
고객의 서비스 이용 만족도, 이용 빈도, 개선 의견을 수집하는 온라인 설문조사 시스템

### 주요 특징
- ✅ 동적 질문 흐름 (branching logic)
- ✅ 다양한 입력 방식 (객관식, 텍스트, 음성)
- ✅ 음성 프롬프트 지원 (접근성)
- ✅ 자동 저장 (진행 중단 복구)
- ✅ 모든 기기 대응 (반응형)
- ✅ 실시간 진행도 표시

### 조사 기본 정보
- **조사ID**: CS2024002
- **조사명**: 2024년 고객 경험 개선을 위한 설문조사
- **기간**: 2024.01.15 ~ 2024.02.15
- **시간**: 09:00 ~ 18:00 (점심 12:00~13:00 제외)
- **담당부서**: 고객만족경영팀
- **예상 소요시간**: 약 15분
- **총 문항**: 7개

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│ 시작 화면 → 로그인/인증 → 설문 소개 → 질문 1~7 → 완료 화면 │

```

### 분기 로직 상세
```
Q1 (연령대) → Q2 (이용 빈도)
                ├─ 주 1회 이상 (1) → Q3 (만족도) → Q6 (개선항목)
                ├─ 월 2-3회 (2) → Q3 (만족도) → Q6 (개선항목)
                ├─ 월 1회 (3) → Q3 (만족도) → Q6 (개선항목)
                ├─ 2-3개월 (4) → Q4 (저이용 이유) → Q6 (개선항목)
                └─ 거의 미사용 (5) → Q5 (저이용 음성) → Q6 (개선항목)
                
Q6 (개선항목) → Q7 (최종 의견) → 완료
```

## 기능 설명

### 질문 구조
각 질문은 다음과 같은 필드로 구성됩니다:
- **questionId**: 질문 고유 ID (Q1, Q2, ...)
- **title**: 질문의 짧은 제목 (예: "연령대", "서비스 이용 빈도")
- **prompt**: 실제 질문 텍스트
- **questionType**: 질문 유형 (객관식, 자유텍스트, 음성)
- **promptType**: 프롬프트 방식 (텍스트, 음성)

### 7개 질문 구성

| 질문ID | title | questionType | promptType | 필수 |
|--------|-------|----------|-----------|------|
| Q1 | 연령대 | multiple_choice | text_prompt | ✓ |
| Q2 | 서비스 이용 빈도 | multiple_choice | voice_prompt | ✓ |
| Q3 | 서비스 만족도 | multiple_choice | text_prompt | ✓ |
| Q4 | 저이용 사유 | text_opinion | text_prompt | ✗ |
| Q5 | 미이용 사유(음성) | voice_opinion | voice_prompt | ✗ |
| Q6 | 개선 우선순위 | multiple_choice | text_prompt | ✗ |
| Q7 | 최종 의견 | voice_opinion | voice_prompt | ✗ |

### 동적 질문 흐름 (Branching Logic)
```json
Q2 응답 값에 따라 다음 질문 결정:
- 값 "1", "2", "3": Q3으로 이동 (고이용자)
- 값 "4": Q4로 이동 (저이용자 - 텍스트)
- 값 "5": Q5로 이동 (미이용자 - 음성)
```

### 자동 저장 기능
- 3초마다 자동 저장
- 현재 답변 저장
- 중단된 설문 복구 기능
- 브라우저 세션 스토리지 + 백엔드 동기화

### 진행도 표시
```
■■■□□ 3/7 (42%)
- 프로그레스 바로 시각적 표현
- 현재 문항 번호 표시
- 완료율 백분율 표시
```

### 유효성 검증
- 필수 항목 확인
- 텍스트 길이 검증 (최소 10자)
- 음성 녹음 시간 검증 (5~120초)
- 선택 개수 검증 (1~3개)

### 접근성 (Accessibility)
- ✅ 키보드 네비게이션
- ✅ 스크린 리더 지원 (aria 라벨)
- ✅ 음성 프롬프트 자막 제공
- ✅ 명도 대비 WCAG AA 준수
- ✅ 포커스 표시 명확
